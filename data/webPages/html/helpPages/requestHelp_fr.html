Champs obligatoires

Champs Définitions.
Type de fichiers Permet de spécifier si on doit traiter des RX ou des TX.
Machines Permet de specifier de quelle(s) machine(s)les données concernant le client ou la source proviennent.
Types de données Permet de specifier pour quel type de donner nous produirons les graphiques.
Laps de temps pré-déterminés Au lieu de specifier un nombre d'heures, utilisez cette option pour utilisés des laps pré-calculés.
Laps de temps fixe Lorsque vous utilisez des laps de temps pré-déterminés, utilisez cette option pour obtenir des intervales de temps réels. Exemple : l'utilisation de l'option mensuel et de l'option mois précédent permettront de créer un graphique débutant le premier du mois précédent et se terminant le premier jour du mois courant.
Sourlients Permet de spécifier pour quelle source ou client(sourlient) les graphiques devront être produits. Pour se faire, veuillez utiliser la fenêtre qui apparaîtra en cliquant sur le bouton "Ajouter". La liste de choix offerts dépendra de votre choix de type de fichier et de machine.

Options avancées (Champs optionels)

Champs Définitions.
Cacher les options avancées Cacher les options avancées va annuler tout choix d'options effectué dans la section des options avancées.
Temps de fin Cette option permet de specifier une date de fin du graphique, ou la date de sa requête. Pour se faire, vous pouvez choisir une date de fin grâce au calendrier prévu à cet effet ou remplir la boîte de texte manuellement, en respectant le format de date authorisé.
Laps de temps Permet de specifier le laps de temps présenté dans les graphqiues. Le nombre spécifié représentera le nombre d'heures voulues. Si aucune valeur n'est spécifié, et qu'aucun laps pré-déterminé n'est utilisé, la valeur 24 sera utilisée.
Produits Permet de spécifier une liste de produits particuliers. Les données utilisées pour produire les graphique proviendront donc seulement des données relatives au produits spécifiés. Note : Les noms de produits peuvent contenir des caractères de type "wildcard".
Combiner les sources ou clients En utilisant cette options, les données de toutes les sources ou clients seront jumelées de manière à ne paraître que comme une seule entité.
