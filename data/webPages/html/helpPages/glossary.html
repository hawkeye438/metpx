Help on the terms employed in pxStats.

Term Definition.
Bytecount Number of bytes that a received or transmitted file weights. In graphics we are interested in the total amount of bytes per x amount of time, not the average.
Client Entity to wich px transmits data.
Errors The total amount of error lines that were found within log files during x amount of time.
Filecount The total amount of files that were received or transmitted during x amount of time.
Latency Delay of time between the time of reception of a file and the time of transmission a file to a specific client. Latencies displayed in graphics are the average latency over x amount of time.
Machine Machine on wich PX is running and producing log files. These log files are the source of the data that is displayed in the graphics. Machine examples : PXATX, PDS5,PDS6
Latency over 15 seconds. 15 seconds is a treshold that has been accepted for acceptable latencies. Thus latencies over 15 seconds need to be watched.
Px Product eXchange. This is the software for wich the graphics are created. This software handles receptions(RX) and transmissions(TX) of meteorological data files.
RX Rx in pxStats means reception. RX graphics display information on the receptions that were made using PX on a specific machine.
Source Entity from wich px receives data.
TX TX in pxStats means transmissions. TX graphics display information on the receptions that were made using PX on a specific machine.
