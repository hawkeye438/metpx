Aide sur les termes employés dans le logiciel pxStats.

Terme Définition.
Nombre de bytes Mesure l'espace disque occupé par un fichier qui a été reçu ou envoyé. Pour les graphiques, nous nous intéressons à la quantité totale envoyé ou reçu durant une période donnée, et non la moyenne de cette période.
Client Entité à laquelle pxStats transmet des données.
Erreurs Le nombre total de lignes d'erreurs se trouvant dans les fichiers de journalisation durant une période donnée.
Nombe de fichiers Le nombre total de fichiers reçu ou transmis pendant une certaine durée.
Latence Délai entre la réception d'un fichier et le temps de réception du fichier de la part du client. Les latences affichées sur les graphiques montre la latence moyenne sur une prédiode de temps donnée.
Machine Machine sur laquelle metPX est exécutée et qui produit des fichiers de journalisation. Ces fichier sont la source des données affichées par les graphiques
Latences au dessus de 15 secondes. 15 secondes est le temps maximale allouée pour qu'une trasmission soit jugée acceptable. Au delà de 15 secondes, ces transmissions doivent être notée de façon à cerner des anomalies avec le logiciel.
Px Product eXchange. Ce logiciel est le logiciel pour lequel les graphiques sont produits. Ce logiciel contrôle les réceptions(RX) and transmissions(TX) de données météorologiques.
RX Rx signifie réception. Les graphiques de type rx n'affichent que les données relatives au réceptions de fichiers.
Source Entité qui envoit des fichiers à px.
TX TX signifie transmission. Les graphiques de type TX n'affichent que les données relatives au transmissions de fichiers.
